1. Business Requirements
1.1. Передумови

Природні катастрофи, такі як землетруси, повені, лісові пожежі та виверження вулканів, загрожують життю людей і спричиняють значні економічні збитки. У сучасному світі критично важливо мати оперативну систему моніторингу, яка дозволяє в реальному часі відстежувати події, аналізувати їх та інформувати користувачів для зменшення наслідків катастроф.
GeoGuard Pro – це інноваційне рішення, яке забезпечує глобальний моніторинг та аналітику природних катастроф за допомогою інтеграції різних джерел даних. 

1.2. Business Opportunity
GeoGuard Pro дозволяє:

1. Оперативно відстежувати природні катастрофи у режимі реального часу.
2. Інформувати населення про загрози.
3. Надавати інструменти для аналізу ризиків.

1.3. Business Objectives
Основні бізнес-цілі:

1. Забезпечити стабільну роботу системи моніторингу з високою точністю даних.
2. Зменшити час реакції на критичні події.
3. Надати користувачам зручний інтерфейс для перегляду інформації про катастрофи.


1.4. Customer or Market Needs
Потреби ринку та клієнтів:


1. Наявність інтегрованої платформи для моніторингу природних катастроф.
2. Швидке та точне відображення подій на інтерактивній карті.
3. Інформування про критичні події через візуальні індикатори, графіки та сповіщення.
4. Доступ до детальної статистики для аналізу та прийняття рішень.
5. Зручна система налаштувань для користувача, що дозволяє адаптувати інтерфейс під особисті вимоги.


1.5. Business Risks
Основні бізнес-ризики:


1. Нестабільність зовнішніх API (USGS, EONET, IP-геолокаційні сервіси), що може вплинути на точність даних.
2. Конкуренція з боку інших систем моніторингу катастроф.
3. Технічні проблеми при інтеграції різних джерел даних.
4. Ризик невідповідності очікуванням користувачів через швидко змінювані вимоги ринку.


2. Vision of the Solution
2.1. Vision Statement

GeoGuard Pro – це інноваційна глобальна система моніторингу природних катастроф, що забезпечує оперативне відстеження, аналіз та інформування про критичні події у режимі реального часу. Завдяки інтеграції декількох API та сучасним технологіям візуалізації, система допомагає урядовим установам, екстреним службам та звичайним користувачам приймати швидкі та ефективні рішення для запобігання та мінімізації наслідків катастроф.


2.2. Major Features
1. MF-1: Інтерактивна карта подій
Ця функція дозволяє користувачам переглядати природні катастрофи в реальному часі на глобальній карті. Використовується технологія Leaflet для швидкої та зручної навігації.
2. MF-2: Моніторинг природних катастроф
Система автоматично збирає дані про землетруси, повені, виверження вулканів та пожежі з авторитетних джерел, таких як USGS та EONET, що гарантує точність інформації.
3. MF-3: Графічний аналіз даних
Дані відображаються у вигляді динамічних графіків та таблиць, що допомагає користувачам швидко аналізувати тенденції та визначати потенційні загрози.
4. MF-4: Реальні сповіщення
Користувачі отримують оповіщення про небезпечні події через email, а також push-нотифікації, що забезпечує своєчасне реагування.
5. MF-5: Аналіз місцевості користувача
Інформація про мережу: система ініціалізує геолокацію користувача, а також відслідковує найближчу загрозу від поточної геолокації користувача.


2.3. Assumptions and Dependencies
Припущення:
   1. Зовнішні API будуть доступні та стабільно працюватимуть.
   2. Користувачі матимуть доступ до інтернету з достатньою швидкістю для отримання даних у реальному часі.
   3. Система буде використовуватись переважно у веб-середовищі з сучасними браузерами.


Залежності:
   1. API USGS, EONET для отримання даних про катастрофи.
   2. API для отримання інформації про IP та геолокацію (ipapi.co).
   3. Бібліотеки JavaScript: Leaflet, Chart.js, AOS, Font Awesome.
   4. Серверна інфраструктура для розміщення веб-додатку (хостинг, CDN).

3. Scope and Limitations
3.1. Серверна частина

Серверна частина системи GeoGuard Pro відповідає за отримання, обробку та передачу даних про глобальні катастрофи. Вона взаємодіє з API OpenWeather, EONET (NASA) та USGS, щоб отримувати актуальну інформацію про стихійні лиха, такі як землетруси, повені, вулкани та пожежі. Отримані дані проходять фільтрацію відповідно до вибраного періоду спостереження, який може становити 3, 7 або 30 днів, а також аналізуються на відповідність критичним рівням загрози. Для землетрусів враховується поріг магнітуди, а для інших катастроф оцінюється рівень активності.  
Крім загального аналізу, система проводить геолокаційний аналіз, визначаючи найближчі події відносно місцезнаходження користувача. Оновлення інформації відбувається автоматично кожні 5 хвилин або за запитом користувача. Однак робота сервера залежить від доступності сторонніх API, що може спричиняти затримки в оновленні даних. Точність геолокації визначається на основі IP-адреси, що також може впливати на коректність результатів.


3.2 Scope of Subsequent Releases
Веб-клієнт:
Поточна версія веб-клієнта GeoGuard Pro надає систему глобального моніторингу катастроф. Вона включає в себе інтерактивну карту з підтримкою даних про землетруси, повені, виверження вулканів та лісові пожежі. Додаткові можливості включають фільтрування подій, статистичний аналіз з візуалізацією через Chart.js та систему критичних сповіщень.
Користувачі можуть перемикатися між світлою і темною темами, налаштовувати параметри сповіщень та відстежувати зміни у режимі реального часу. Інформація отримується з відкритих API, таких як NASA EONET та USGS, а карта базується на OpenStreetMap через бібліотеку Leaflet. Оновлення даних відбувається кожні 5 хвилин.
3.3. Limitations and Exclusions

Система вимагає постійного підключення до інтернету, оскільки офлайн-доступ до даних не передбачено. Наразі підтримка локальних сповіщень (push notifications) не реалізована, тому користувачі отримують оновлення лише під час активного використання веб-клієнта.
Також інтерфейс оптимізований переважно для десктопних браузерів, і хоча мобільна версія доступна, її функціональність може бути обмеженою.


4. Business Context
4.3. Operating Environment
Операційне середовище:
   1. Географічне розташування користувачів: Користувачі можуть бути розкидані по всьому світу, тому система повинна бути оптимізована для роботи в різних часових зонах.
   2. Час доступу: Система повинна бути доступною 24/7, оскільки надзвичайні події можуть траплятися у будь-який час.
   3. Інфраструктура: Дані генеруються з різних джерел (API USGS, EONET, IP-геолокаційні сервіси) та комбінуються на стороні клієнта; обробка відбувається в браузері користувача.
   4. Продуктивність: Система повинна забезпечувати швидке завантаження карти, оновлення даних кожні 5 хвилин, а також швидке реагування на дії користувача.
   5. Безпека: Забезпечення захисту даних користувача, зокрема інформації про IP та геолокацію, відповідно до сучасних стандартів безпеки.
   6. Сумісність: Підтримка сучасних веб-браузерів (Chrome, Edge, Brave) з адаптивним дизайном для різних пристроїв.
